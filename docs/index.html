Instalación

k6-httpx es un módulo local. Solo copia el archivo en tu proyecto.

Estructura del proyecto
mi-proyecto/
├── helpers/
│   └── k6.httpx.js      ← la librería
├── examples/
│   ├── 01_pokemon_basics.js
│   └── ...
└── script.js

Requisito: k6 versión 0.45+ instalado. Instalar k6 →
script.js javascript 
// Importar desde la ruta relativa
import { httpx } from "./helpers/k6.httpx.js";
import { sleep } from "k6";

// Init context — configuración global
httpx.baseUrl("https://api.miapp.com");

export default function () {
  httpx.get("/users").expectStatus(200);
  sleep(1);
}
Ejecutar
$ k6 run script.js
$ k6 run script.js --vus 10 --duration 30s
$ k6 run script.js -e BASE_URL=https://staging.api.com
Quickstart

Tu primer test en 5 minutos

Un ejemplo completo usando la PokéAPI pública.

quickstart.js javascript 
import { httpx } from "./helpers/k6.httpx.js";
import { sleep } from "k6";

// ① Init context: configurar base URL y declarar métricas
httpx.baseUrl("https://pokeapi.co/api/v2");
httpx.declareMetric("pokemon_ms");

export const options = {
  thresholds: {
    http_req_duration: ["p(95)<2000"],
    pokemon_ms:        ["p(95)<1500"],
  }
};

// ② Flujo VU
export default function () {

  httpx.step("Buscar Pokémon", () => {

    const res = httpx.get("/pokemon/pikachu")
      .expectStatus(200)          // ③ assert status
      .expectTime(2000)           // ④ assert duración
      .contract({                  // ⑤ validar tipos
        id:   "number",
        name: "string",
      })
      .extract({ pid: "id" });    // ⑥ extraer valor

    httpx.metric("pokemon_ms", res.duration());
    console.log(`ID: ${httpx.var("pid")}`);
  });

  sleep(1);
}
API Reference

HTTP Methods

MétodoFirmaDescripción
httpx.get()(url, params?)Petición GET. Añade sesión y cookie jar automáticamente.
httpx.post()(url, body?, params?)Petición POST con body (string o object).
httpx.put()(url, body?, params?)Actualización completa de un recurso.
httpx.patch()(url, body?, params?)Actualización parcial de un recurso.
httpx.delete()(url, params?)Eliminar un recurso.
httpx.head()(url, params?)Verificar disponibilidad sin descargar body.
httpx.parallel()(requests[])Batch de requests concurrentes via http.batch().
Nota: Todos los métodos retornan un HttpxResponse con API fluent. Puedes encadenar assertions: .expectStatus(200).expectTime(500).contract({...})
API Reference

Session / Auth

Gestión de tokens y headers globales aplicados a todos los requests.

MétodoFirmaDescripción
httpx.session.setToken()(token, header?, prefix?)Añade Authorization: Bearer {token} a todos los requests.
httpx.session.setHeader()(name, value)Añade un header global personalizado.
httpx.session.clearToken()()Elimina el token y los headers extra.
auth-example.jsjavascript
// En beforeAll o setup()
const res = httpx.post("/auth/login", JSON.stringify({ user: "x", pass: "y" }),
  { headers: { "Content-Type": "application/json" } });

// Token aplicado a TODOS los requests siguientes
httpx.session.setToken(res.json().token);

// Header personalizado global
httpx.session.setHeader("X-Tenant-Id", "acme-corp");

// Token API Key en header custom
httpx.session.setToken(apiKey, "X-Api-Key", "");
API Reference

Assertions

Métodos de verificación encadenables sobre el objeto HttpxResponse.

MétodoFirmaDescripción
.expectStatus()(code: number)Verifica que el status sea exactamente code.
.expect2xx()()Verifica que el status esté en el rango 200–299.
.expectTime()(ms: number)Verifica que la duración sea menor a ms milisegundos.
.expectHeader()(name, value)Verifica que el header name contenga value.
.expectJSON()(expected: object)Verifica pares clave-valor en el JSON. Soporta dot-notation.
.expectJSONSchema()(schema: object)Verifica tipos de campo. No lanza excepción (usa check()).
.expectXML()(tags: string[])Verifica que el body XML contenga las etiquetas indicadas.
.softExpect()(fn)Ejecuta assertions sin detener el test si fallan (warning).
API Reference

Contract Validation

Valida la forma del JSON de respuesta con un esquema de tipos.

MétodoFirmaDescripción
.contract()(schema: object)Si la respuesta es array, valida el primer elemento. Si es objeto, lo valida directamente. Lanza excepción si falla.
.contractArray()(schema: object)Valida que la respuesta sea un array y que cada elemento cumpla el esquema.

Tipos soportados en el schema:

"string" "number" "boolean" "object" "array" "null" "any"
contract-example.jsjavascript
// GET /users → [{ id: 1, name: "Alice" }, ...]
httpx.get("/users")
  .contract({ id: "number", name: "string" });  // valida users[0] ✔

// GET /users/1 → { id: 1, name: "Alice" }
httpx.get("/users/1")
  .contract({ id: "number", name: "string" });  // valida objeto ✔

// Validar TODOS los elementos del array
httpx.get("/users")
  .contractArray({ id: "number", email: "string" }); // todos los items ✔

// Dot-notation para propiedades anidadas
httpx.get("/profile")
  .contract({ "address.city": "string", "address.zip": "string" });
API Reference

Extract / Correlación

Extrae valores de la respuesta para usarlos en requests posteriores.

MétodoFirmaDescripción
.extract()(map: object)Guarda valores del JSON en el correlation store. Soporta dot-notation y array index.
httpx.var()(name: string)Lee un valor previamente guardado con .extract().
extract-example.jsjavascript
// Paso 1: extraer valores del POST /login
httpx.post("/login", payload)
  .extract({
    authToken:  "token",             // res.token
    userId:     "user.id",           // res.user.id (dot-notation)
    firstItem:  "[0].id",            // array[0].id
    nestedItem: "items[2].name",     // res.items[2].name
  });

// Paso 2: usar los valores en el request siguiente
httpx.session.setToken(httpx.var("authToken"));
httpx.get(`/users/${httpx.var("userId")}`).expectStatus(200);
API Reference

Métricas Custom

Trend metrics personalizados con tags, visibles en el resumen final de k6.

Regla de oro: httpx.declareMetric() SIEMPRE en el init context (nivel raíz del script). httpx.metric() se puede llamar dentro de default().
MétodoContextoDescripción
httpx.declareMetric(name)Init context ⚠️Registra un nuevo Trend metric. Llámalo al nivel raíz del script.
httpx.metric(name, value, tags?)VU context ✅Registra un valor al metric declarado. Acepta tags para filtrar en Grafana.
metrics-example.jsjavascript
// ✅ CORRECTO — declareMetric en init context
httpx.declareMetric("checkout_ms");
httpx.declareMetric("search_ms");

export const options = {
  thresholds: {
    checkout_ms: ["p(95)<500"],  // thresholds sobre la métrica custom
    search_ms:   ["p(90)<300"],
  }
};

export default function () {
  const res = httpx.get("/search?q=test");

  // Registrar con tags para filtrar por endpoint/región en Grafana
  httpx.metric("search_ms", res.duration(), {
    endpoint: "search",
    region:   "us-east",
  });
}

Parallel

Requests concurrentes usando http.batch() de k6 internamente.

parallel-example.jsjavascript
// Formato: [method, url, body?, params?]
const [r1, r2, r3] = httpx.parallel([
  ["GET",  "/users"],
  ["GET",  "/products"],
  ["POST", "/events", JSON.stringify({ type: "pageview" }),
    { headers: { "Content-Type": "application/json" } }],
]);

r1.expectStatus(200);
r2.expectStatus(200);
r3.expectStatus(201);

Retry con Backoff Exponencial

OpciónDefaultDescripción
retries3Número máximo de intentos.
baseDelay1Segundos de espera en el primer reintento.
factor2Multiplicador del delay por intento.
maxDelay30Cap máximo de segundos de espera.
retry-example.jsjavascript
// Delays: 0.5s → 1s → 2s → 4s (capped a maxDelay)
httpx.retry(
  () => {
    const res = httpx.get("/health");
    if (res.status() !== 200) throw new Error(`status: ${res.status()}`);
    return res;
  },
  { retries: 4, baseDelay: 0.5, factor: 2, maxDelay: 10 }
);
API Reference

Lifecycle Hooks

Funciones que se ejecutan una vez antes/después de todas las iteraciones.

hooks-example.jsjavascript
// Registrar hooks en init context
httpx.beforeAll(() => {
  // Se ejecuta en setup() — una vez antes de todas las VUs
  const res = httpx.post("/auth/login", credentials);
  httpx.session.setToken(res.json().token);
});

httpx.afterAll(() => {
  // Se ejecuta en teardown() — después de todas las VUs
  httpx.printTimeline();
  httpx.session.clearToken();
});

// En las funciones de k6
export function setup()    { httpx.runHooks("beforeAll"); }
export function teardown() { httpx.runHooks("afterAll");  }

Guía k6

Init Context vs VU Context

La regla más importante de k6: algunos recursos solo pueden crearse al inicio.

Init context = código a nivel raíz del módulo (fuera de cualquier función).
VU context = dentro de default(), setup(), teardown().
contextos.jsjavascript
import { httpx } from "./helpers/k6.httpx.js";

// ✅ INIT CONTEXT — se ejecuta UNA VEZ al cargar el script
httpx.baseUrl("https://api.com");       // ✅
httpx.declareMetric("my_ms");           // ✅ new Trend() aquí
httpx.beforeAll(() => { /* ... */ });   // ✅ registrar hook
httpx.use(myPlugin);                    // ✅ registrar plugin

// ❌ NUNCA en init context:
// httpx.get("/anything")              ← requests en init context
// httpx.cookies                       ← cookie jar en init context

export default function () {
  // ✅ VU CONTEXT — se ejecuta por cada iteración de cada VU
  httpx.get("/users");                  // ✅ requests aquí
  httpx.metric("my_ms", 123);          // ✅ registrar valor
  const jar = httpx.cookies;           // ✅ cookie jar aquí
}
